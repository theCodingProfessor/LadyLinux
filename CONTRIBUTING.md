# Contributing to Lady Linux

Lady Linux welcomes contributors of all backgrounds.

You do not need to be a kernel developer to participate.

---

## Ways to Contribute

### Technical
- Python development
- Security review
- LLM experimentation
- System testing
- Virtualization experiments

### Non-Technical
- Documentation
- UX feedback
- Educational content
- Ethics and data rights review
- Writing and editing

---

## Contribution Principles

- Respect human agency
- Favor inspectability over convenience
- Document decisions
- Assume good intent
- Keep humans in the loop

---

## Getting Started

1. Read `VISION.md`
2. Review `ARCHITECTURE.md`
3. Open an issue with questions or ideas
4. Propose small, scoped contributions first

---

## Code of Conduct

Be respectful.
Be curious.
Be transparent.
